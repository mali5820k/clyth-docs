Strings and characters

Clyth 0.5.0 supports string values, string literals, formatted literals, characters, and array types used by the released runtime and sample programs.

Current contract: this page documents the released surface. Proposed value-copy rules, automatic string destruction, interning, and MECC-backed storage are future design work.

String values

Use string for text accepted by runtime APIs and formatted literals. The 0.5.0 compiler and runtime provide the representation used by current modules; user code should not assume a future ownership or storage optimization.

string project = "Clyth"
string release = `${project} 0.5.0 Alpha`
println(release)

Characters and arrays

char represents an individual character. Array syntax may be fixed-size or dynamic according to the declared type and the compiler/runtime path being used.

char marker = 'C'
char[5] name = ['C', 'l', 'y', 't', 'h']

Formatted literals

Backtick literals interpolate expressions with ${...}. This behavior is covered by the 0.5.0 regression suite.

int32 passed = 24
println(`regression tests: ${passed}/24`)

Future design direction

The self-hosted compiler is expected to define richer value semantics, scope-aware cleanup, char-array conversion and slicing rules, compiler-materialized temporary storage, and possible interning or MECC optimizations. Those ideas are intentionally not presented as implemented 0.5.0 behavior.
