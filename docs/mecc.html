MECC

MECC means Managed Entanglement for Collapsible Collections. The name describes the intended ownership model: allocations are not only individual resources, but can be part of related estate graphs whose cleanup can collapse deterministically when a root estate is released.

Root estate collapse

An estate is a lifetime region. Estates may depend on other estates. When a root estate loses ownership, dependent estates can decrement and collapse through known cleanup paths.

RootEstate app
  ├─ Estate session
  │    ├─ JsonDocument body
  │    └─ List<string> messages
  └─ Estate cache
       └─ Map<string, string>

app.clear()
// session and cache collapse when their dependency counts reach zero

Not tracing garbage collection

MECC is not intended to scan the entire heap looking for reachable objects. It works over known ownership relationships, explicit resources, and compiler-visible cleanup paths.

// Manual foundation in 0.5.x
request_scope.free()

// Future MECC direction
mecc estate request_scope {
    JsonDocument body = JSON.parse(request.body())
    response.send(200, body.to_string())
}
// compiler collapses request_scope here

Connection to free()

free() is the primitive cleanup operation. MECC builds on that convention by grouping resources into estates and reducing repetitive cleanup code where ownership is clear.

struct Session {
    JsonDocument body
    List<string> tags
}

void Session.free() {
    this.body.free()
    this.tags.free()
}
