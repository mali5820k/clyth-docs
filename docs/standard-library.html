Standard Library and Runtime Modules

The 0.5.0 runtime surface is intentionally practical: file I/O, printing, JSON, hashing, collections, and a server abstraction. Each module follows the wrapper → ABI → native provider model rather than exposing native library layouts directly.

File I/O and printing

The file I/O module provides synchronous user-facing calls backed by the native file subsystem. In 0.5.0 the backend is routed through libuv. libuv itself uses a default worker pool of four threads for filesystem operations. That default is suitable for moderate workloads, but server-heavy or file-heavy applications may eventually need runtime configuration for the worker pool size and scheduling policy.

FunctionDescription
void print(string value)Writes a value to standard output without appending a newline. The value should be a string today; later to_string() integration can generalize this.
void println(string value)Writes a value to standard output and appends a newline.
string read_file(string path)Reads the full contents of a file and returns it as a Clyth string.
void write_file(string path, string contents)Writes contents to a file, replacing existing contents.
void append_file(string path, string contents)Appends contents to a file.
bool file_exists(string path)Returns true when the path exists.
void delete_file(string path)Deletes the file at the given path when supported by the backend.

File I/O usage

Use file I/O through Clyth names, not through direct POSIX calls. That allows the native provider to remain libuv-backed today and evolve later without changing application code.

include "file-io"

int32 main(string[] args) {
    write_file("message.txt", "hello from Clyth")

    if file_exists("message.txt") {
        string text = read_file("message.txt")
        println(`file contents: ${text}`)
        delete_file("message.txt")
    }

    return 0
}

JSON

The JSON module is backed by yyjson through a thin ABI layer. Clyth owns the JSON surface and JsonValue concepts. yyjson provides parsing/stringify performance below the ABI boundary.

MethodDescription
bool JSON.is_valid(string text)Validates whether a string is parseable JSON.
JsonValue JSON.parse(string text)Parses a JSON string and returns a runtime JSON value handle/wrapper.
string JSON.stringify(JsonValue value)Converts a supported value or JSON value into a JSON string.
string JsonValue.to_string()Returns the serialized representation of the JSON value.

JSON with a route handler

JSON commonly appears at server boundaries. Request bodies arrive as strings; the JSON module validates or parses them; responses can be serialized with JSON.stringify.

include "https"
include "json"

router.post("/api/message", (Request request, Response response) => {
    string body = request.body()

    if JSON.is_valid(body) {
        response.send(200, JSON.stringify({ "received": true }))
    } else {
        response.send(400, JSON.stringify({ "error": "invalid JSON" }))
    }
})

Hashing

The hash module uses rapidhash as the native provider. Hashing is runtime functionality: the compiler does not need to know how rapidhash works. It only links the module and exposes Clyth wrapper functions.

FunctionDescription
uint64 hash_string(string value)Hashes a string and returns a 64-bit value.
uint64 hash_bytes(uint8[] data)Hashes byte-oriented data when the runtime representation is available.
HasherA future wrapper direction for incremental hashing and seeded hashing.

Collections

Collections are generic runtime types. They are not intended to be a special compiler-only feature. The compiler checks syntax and types; the runtime module owns behavior, allocation strategy, and hash integration.

FunctionDescription
List<T>Ordered dynamic sequence of values of type T.
Set<T>Uniqueness-oriented collection backed by hashing support.
Map<K,V>Key-value collection using hash support for lookup.

Generic collection usage

Generic collection syntax keeps type information visible at the call site while leaving storage behavior in the runtime module.

include "collections"

List<int32> ports = List<int32>()
ports.push(8080)
ports.push(9090)

Map<string, int32> status = Map<string, int32>()
status.set("ok", 200)
status.set("not_found", 404)

HTTPS / server module

The server-facing module is https. Plain HTTP is the behavior when TLS certificates are not configured; TLS becomes a configuration detail rather than a separate user-facing server type. The native implementation uses libuv for the event loop and TCP behavior, llhttp for HTTP parsing, and OpenSSL for TLS direction.

FunctionDescription
Router()Creates a route registry.
void Router.get(string path, function<void(Request, Response)> handler)Registers a GET route.
void Router.post(string path, function<void(Request, Response)> handler)Registers a POST route.
string Request.body()Returns the request body associated with the current native request handle.
string Request.method()Returns the HTTP method.
string Request.path()Returns the parsed request path.
string Request.query()Returns query-string data when present.
void Response.send(int32 status, string payload)Sends an HTTP response with a status code and payload.
ServerConfiguration(int32 port, Router router)Configures server startup for the current runtime.
int32 Server.start()Starts the libuv event loop and serves until shutdown.

Server usage

The route handler is an ordinary Clyth lambda whose parameters are wrapper objects. The callback ABI remains stable because Request and Response carry handles rather than native parser/socket structs.

include "https"
include "json"

int32 main(string[] args) {
    Router router = Router()

    router.get("/", (Request request, Response response) => {
        response.send(200, "<h1>Hello from Clyth</h1>")
    })

    router.post("/api/message", (Request request, Response response) => {
        println(`body: ${request.body()}`)
        response.send(200, JSON.stringify({ "received": true }))
    })

    ServerConfiguration configuration = ServerConfiguration(9090, router)
    Server server = Server(configuration)
    server.start()

    return 0
}
