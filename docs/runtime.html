Clyth 0.5.0

Runtime Architecture

The central 0.5.0 architecture is the separation between Clyth-facing wrappers and native implementation details.

Layering

A Clyth module should expose ordinary Clyth values and methods. Native libraries remain below the ABI boundary. This keeps user code readable and keeps runtime implementation details replaceable.

Clyth application
    ↓
Clyth wrapper module
    ↓
Stable C ABI boundary
    ↓
Native runtime provider
    ↓
libuv / yyjson / rapidhash / OS

Why handles exist

Exposing raw C structs would couple Clyth source to native layout. A handle keeps the layout private. Clyth code can call methods on the wrapper, while the native provider can store whatever state it needs internally.

request.body()
    ↓
Request.body(request)
    ↓
clyth_https_request_body(request.handle)
    ↓
lookup request state in native runtime
    ↓
return Clyth string
