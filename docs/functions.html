Functions and callbacks

Functions declare a return type, name, typed parameters, and a body. Clyth 0.5.0 also supports first-class function values and lambda expressions used by the runtime APIs.

Ordinary functions

Parameters in the released compiler follow its current tested calling behavior. The proposed rule where named arguments become mutable aliases is not part of 0.5.0.

int32 add(int32 left, int32 right) {
    return left + right
}

int32 total = add(20, 22)

First-class function values

The canonical function type places the return type before its parameter list.

function<int32(int32, int32)> operation = add
int32 result = operation(3, 4)

Lambdas

Lambdas use => and can be passed into callback-oriented APIs.

function<void(int32)> report = (int32 value) => {
    println(`${value}`)
}

report(24)

Future parameter semantics

Mutable alias parameters and compiler-materialized temporary arguments were explored after the 0.5.0 baseline. Their final form will be implemented and stress-tested in the self-hosted compiler rather than claimed by this release.
