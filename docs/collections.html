Collections

Collections are runtime types, not compiler-owned syntax tricks. They are generic Clyth-facing APIs backed by runtime storage and hashing where appropriate.

List

List<T> stores ordered values. As of 0.5.x, constructor types are explicit. Constructor inference such as List<int32> values = List() is a 0.6.0 compiler simplification target.

include "collections"

List<int32> values = List<int32>()
values.add(10)
values.add(20)

println(`${values.get(0)}`)
values.free()

Map

Map<K,V> maps keys to values. The duplicated generic constructor form is current 0.5.x syntax; simplifying it belongs with the bootstrapped compiler rewrite.

Map<string, int32> scores = Map<string, int32>()
scores.set("compiler", 1)
scores.set("runtime", 2)

println(`${scores.get("runtime")}`)
scores.free()

Set

Set<T> stores unique values. Hashing is provided through the hash module/runtime implementation.

Set<string> names = Set<string>()
names.add("Clyth")
names.add("MECC")

if names.contains("Clyth") {
    println("found")
}
names.free()
