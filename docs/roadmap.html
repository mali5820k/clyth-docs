Roadmap

The roadmap follows the same principle as the runtime architecture: keep the language identity above any one implementation dependency while still using mature tooling where it provides real value.

0.5.0 — Runtime Ecosystem Foundation

Lock the current alpha language/runtime surface, validate the Clyth wrapper → ABI → native provider model, and release a clean musl/libc++-audited artifact path.

  • 20/20 regression suite passing.
  • Web backend demo with libuv event loop, router callbacks, Request/Response wrappers, and JSON POST flow.
  • File I/O routed through the native runtime provider.
  • Recursive audit verifying no GNU/glibc/libstdc++ markers in checked release artifacts.

0.6.0 — Compiler Simplification and Bootstrap Path

The next major engineering step is to make the compiler easier to own. That means reducing dependency complexity while preserving the artifact model that 0.5.0 established.

  • Begin rewriting the frontend in Clyth proper.
  • Replace ANTLR with a compact Pratt parser.
  • Introduce Clyth-owned AST and IR structures.
  • Create a Clyth-side LLVM IR emission library/interface.
  • Keep LLVM optimization/code generation available through IR and command-line tools.

Later milestones

  • Standalone x86_64-linux-musl and aarch64-linux-musl compiler artifacts.
  • Expanded module build/distribution tooling.
  • MECC ownership analysis and cleanup insertion.
  • Further reduction of LLVM C++ API dependency.
  • Self-hosted compiler/runtime ecosystem direction.
