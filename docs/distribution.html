Distribution

Distribution is part of the trust story: the artifact should be lean, reproducible, and clear about what it contains.

Lean release artifact

The release tarball should not contain giant source/build trees. Users should receive the compiler, runtime files needed to build programs, samples, docs, and license notices.

clyth-dist.tar.gz
  clyth
  docs/
  sample-clyth-programs/
  clyth-runtime/
  LICENSE.md
  NOTICE.md

Audit before release

The audit step protects the release promise that artifacts are not accidentally tied to glibc or GNU libstdc++.

./audit_no_gnu_abi.sh

Expected:
  [ OK ] No GNU/glibc/libstdc++ ABI markers found
