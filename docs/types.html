Language

Primitive and composite types

Clyth uses explicit-width numeric types alongside booleans, characters, strings, arrays, structs, function values, and runtime collection types.

Integer types

SignedUnsignedWidth
int8uint88 bits
int16uint1616 bits
int32uint3232 bits
int64uint6464 bits

Values overflow or underflow according to their representable width. The 0.5.0 boundary regression verifies unsigned wrapping and widening behavior.

Floating point and boolean

float32 and float64 represent floating-point values. bool represents true or false.

float32 ratio = 0.5
float64 precise = 42.125
bool ready = true

Characters, strings, and arrays

char represents a character, string represents runtime text, and T[N]/T[] forms represent array surfaces supported by the compiler and runtime.

char prefix = 'C'
string name = "Clyth"
int32[] values = [1, 2, 3]

Null

null is available for nullable reference-like or opaque-resource surfaces where the declared type permits absence. Primitive numeric values are not implicitly nullable.

Handle<NativeState> state = null

if state == null {
    println("not initialized")
}

Default initialization

Clyth 0.5.0 Alpha initializes newly declared primitive variables to deterministic values. Numeric values begin at zero, booleans begin as false, and characters begin as the zero character '\0'. The current runtime representation of an uninitialized string is a null string value, which prints as (null).

Type0.5.0 default
int8 through int640
uint8 through uint640
float32 and float640.0
boolfalse
char'\0'
stringnull runtime string; prints as (null)

Observed behavior

This behavior was verified against the released 0.5.0 compiler. Explicit initialization is still recommended when a meaningful application value is available.

Future string work may replace the null runtime representation with an empty value while preserving deterministic initialization.

int32 counter
bool enabled
char separator
float64 total
string message

println(`${counter}`)   // 0
println(`${enabled}`)   // 0 / false
println(`${separator}`) // zero character
println(`${total}`)     // 0.000000
println(`${message}`)   // (null)

Compiler-generated temporaries

A generalized mutable-temporary lowering model is a future compiler design, not part of the 0.5.0 type contract.
