Advanced Language

These are the pieces that matter for module authors and the 0.6.0 bootstrap direction.

Extension methods as organization

Extension methods separate implementation from data layout. They let behavior live outside a struct declaration while duplicate method definitions remain invalid.

struct TestStruct {}

void TestStruct.addTwo(int32 one, int32 two) {
    println(`${one + two}`)
}

Callback ABI discipline

When a callback crosses a native boundary, keep the ABI simple. Pass handles and expose richer behavior through methods.

function<void(Request, Response)> handler = (request, response) => {
    response.send(200, request.body())
}

0.6.0 compiler direction

The goal after 0.5.0 is not more frontend complexity. The goal is to remove it: Pratt parser, Clyth-owned IR, and a Clyth-side LLVM IR generation library.
