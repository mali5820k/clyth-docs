Collections

Clyth 0.5.0 provides generic runtime collections used by application code and by the future self-hosted compiler.

Current syntax: generic constructor arguments remain explicit in the released compiler.

List<T>

List<T> stores ordered values. The collection owns the runtime storage described by its API and exposes explicit cleanup where required.

include "collections"

List<int32> values = List<int32>()
values.add(10)
values.add(20)
println(`${values.length()}`)
values.free()

Map<K, V>

Map<K, V> associates keys with values. Constructor inference may be explored in the self-hosted compiler, but the duplicated generic form is the current 0.5.0 syntax.

Map<string, int32> scores = Map<string, int32>()
scores.set("compiler", 24)
println(`${scores.get("compiler")}`)
scores.free()

Set<T>

Set<T> stores unique values and uses the runtime hashing layer where appropriate.

Set<string> names = Set<string>()
names.add("Clyth")
names.add("MECC")
names.free()

Compiler stress testing

The self-hosted compiler will exercise collections at a larger scale through token streams, syntax nodes, symbol tables, diagnostics, modules, and generated IR. That workload is expected to expose correctness and performance issues that small samples cannot.
