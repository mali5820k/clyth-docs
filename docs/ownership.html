Ownership and free()

free() is Clyth's destructor convention. It gives the compiler one cleanup target to reason about now and gives MECC a predictable cleanup hook later.

Manual cleanup convention

Types that own runtime resources expose free(). Code that allocates owned resources should call free() once ownership is no longer needed. Future compiler passes can insert proven calls automatically.

JsonDocument document = JSON.parse(text)

println(document.to_string())

document.free()

Borrowed handles

Handle<T> represents an opaque runtime reference whose lifetime is owned elsewhere. It is passable through ABI functions, but user code does not free it.

struct Request {
    private Handle<RequestState> native_handle
}

string Request.body() {
    return https_request_body(this.native_handle)
}

Owned handles

OwnedHandle<T> represents an opaque runtime resource owned by the Clyth wrapper. It must have a known release path, usually implemented by the wrapper's free() method.

struct JsonDocument {
    private OwnedHandle<JsonState> native_handle
}

void JsonDocument.free() {
    json_document_release(this.native_handle)
}

Why raw native structs are hidden

Native structs create ABI coupling. If a Clyth struct exposes a libuv or yyjson layout directly, a native dependency update can break the language surface. Opaque resources keep the public API stable while allowing native implementations to evolve.

// Public Clyth wrapper
struct Response {
    private Handle<ResponseState> native_handle
}

void Response.send(int32 status, string payload) {
    https_response_send(this.native_handle, status, payload)
}
