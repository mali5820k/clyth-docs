Types

Clyth uses explicit primitive widths so systems code can state the storage and arithmetic behavior it expects. Numeric overflow and underflow follow the selected integer width; the docs show those boundaries because they are part of the language surface, not an incidental backend detail.

Primitive integer widths

Signed and unsigned integer types are distinct. Use the smallest type that accurately represents the value range, especially in ABI-facing code where field width matters.

int8   a = -128
uint8  b = 255
int16  c = -32768
uint16 d = 65535
int32  e = 2147483647
uint32 f = 4294967295
int64  g = 9223372036854775807
uint64 h = 18446744073709551615

Overflow and underflow

Variables of distinct sizes overflow or underflow as expected when arithmetic exceeds the representable range of that type. These examples are useful regression targets because they make fixed-width behavior visible.

uint8 max = 255
uint8 wrapped = max + 1
// wrapped == 0

uint8 zero = 0
uint8 under = zero - 1
// under == 255

Characters and booleans

char is the scalar character/code-unit type used by fixed character arrays and string conversion APIs. bool is used for branch conditions and API results.

char letter = 'C'
bool ok = true

if ok {
    println(`letter = ${letter}`)
}

Null values

null represents absence for nullable-capable values such as dynamic arrays, collections, and opaque runtime resources. It is not a valid value for numeric primitives.

JsonDocument document = null

if document != null {
    println(document.to_string())
}

// int32 count = null  // semantic error
