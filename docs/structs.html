Structs and Methods

Structs define data layout. Methods define behavior. Clyth supports methods attached inside the struct body and extension-style method declarations outside the struct.

Fields and visibility

Fields are public by default so data layout stays visible. private is used when a field is an implementation detail, such as an ABI handle.

struct Request {
    private Handle handle
}

struct Point {
    int32 x
    int32 y
}

Extension methods

Extension methods allow implementation to be organized separately from layout. They are not a mechanism for replacing existing behavior; duplicate method declarations should be rejected so imports remain predictable.

struct TestStruct {}

void TestStruct.addTwo(int32 one, int32 two) {
    println(`${one + two}`)
}

Wrapper structs

Native-backed modules should expose Clyth structs with methods, not raw native structures. The struct carries a handle. The methods call ABI functions that know how to interpret the handle.

struct Request {
    private Handle handle
}

string Request.body() {
    return clyth_https_request_body(this.handle)
}
