Distribution and Audit

Clyth distribution bundles should remain lean: compiler binary, runtime files, samples, documentation, inspection tools, and license files.

Do not bundle

  • LLVM source trees
  • LLVM build directories
  • Zig toolchains
  • generated object caches
  • unrelated vendor source checkouts
  • temporary tarballs

ABI audit

The release audit rejects accidental GNU/glibc/libstdc++ linkage in checked release artifacts while allowing the musl/libc++ path required by the project.

./audit_no_gnu_abi.sh
