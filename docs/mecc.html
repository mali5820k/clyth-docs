MECC

Managed Entanglement for Collapsible Collections is Clyth's planned estate-oriented ownership model. It is a design target for the path to 1.0, not an implemented 0.5.0 garbage collector or runtime promise.

Managed entanglement

Resources are grouped into estates with known ownership relationships. Estate metadata represents dependencies between related resource collections instead of treating every allocation as an isolated lifetime problem.

Root estate
  ├─ parser estate
  │    ├─ tokens
  │    └─ syntax nodes
  └─ module estate
       └─ semantic data

Collapsible collections

When a root estate loses its final relationship, its known owned resources can be released and dependent estates decremented. A dependent estate that reaches zero can collapse in turn.

root reaches zero
  → run known cleanup paths
  → release root allocations
  → decrement dependents
  → collapse dependents at zero

Not tracing garbage collection

MECC is not intended to scan arbitrary heap reachability and infer survival globally. It operates on compiler-visible estate relationships and explicit cleanup knowledge.

not: scan the whole heap

instead:
collapse a known ownership graph

Why the self-hosted compiler matters

The compiler itself will provide a demanding testbed for estates: lexer buffers, token collections, syntax trees, semantic models, IR modules, diagnostics, and temporary build data naturally form related lifetime regions.
