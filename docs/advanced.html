Language design

Language design and extension points

This page collects implemented organization features and future compiler directions without mixing research proposals into the current language reference.

Extension methods

A method may be defined separately from its struct declaration to keep data layout concise while preserving a normal type-owned method. Duplicate definitions remain invalid.

struct Counter {
    int32 value
}

void Counter.increment() {
    this.value = this.value + 1
}

Stable callback boundaries

Runtime callbacks should cross native boundaries through stable ABI-safe values or opaque resource wrappers rather than exposing provider-specific C++ objects.

router.post("/", (Request request, Response response) => {
    response.send(200, "ok")
})

Compiler evolution

The long-term frontend will be written in Clyth and organized into readable phases. Planned language changes should be implemented there when extending the transitional compiler would add more complexity than lasting value.
