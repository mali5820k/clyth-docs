Compiler

Compiler overview

Clyth 0.5.0 ships a C++-hosted stage-zero compiler. It parses Clyth, performs semantic analysis, emits LLVM IR, and drives native artifact generation.

0.5.0 status: the released compiler is the stable foundation used to begin self-hosting; it is not itself the self-hosted compiler.

Released pipeline

The current frontend uses ANTLR-generated parsing support and C++ semantic/code-generation passes. LLVM remains the optimization and machine-code backend.

Clyth source
  → ANTLR parser
  → AST and semantic analysis
  → LLVM IR
  → LLVM code generation
  → lld and musl-oriented link
  → native executable

Self-hosting transition

The next compiler will live in a separate bootstrapped-compiler/ subtree. The released compiler builds the first Clyth implementation; that implementation then rebuilds itself and compiles the growing regression suite.

Stage 0: C++ clythc
    builds
Stage 1: Clyth clythc
    rebuilds
Stage 2: Clyth clythc

Reduced dependency surface

The self-hosted compiler is intended to emit textual LLVM IR and use compact backend tools such as llc and lld. It will not require libc++ merely because the transitional compiler was written in C++.

clythc
  → module.ll
  → llc
  → module.o
  → lld + musl
  → program

Why self-hosting matters

Writing the compiler in Clyth is both an implementation strategy and a large-scale language test. It exercises parsing, collections, file I/O, control flow, modules, diagnostics, memory behavior, and repeated compilation far beyond small example programs.
